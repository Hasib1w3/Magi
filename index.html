তোমাকে স্বাগতম, ভবিষ্যতের Developer!

এই ক্লাসটা শুরু হচ্ছে একদম শূন্য থেকে। তুমি আগে কখনো কোড দেখোনি, programming শুনলে ভয় লাগে — এটা সম্পূর্ণ স্বাভাবিক। কিন্তু এই ক্লাসের শেষে তুমি নিজেই বুঝবে যে ব্যাপারটা আসলে কতটা সহজ।

🧠

একটু বলে রাখি: এখানে প্রতিটা শব্দ ভাঙা হবে। কোনো কিছু "ধরে নেওয়া" হবে না। তুমি যদি কখনো মনে করো "এটা কেন?" — সেটাই সঠিক প্রশ্ন। সব কিছুর "কেন" এখানে আছে।

পাঠ ০১

প্রোগ্রামিং আসলে কী জিনিস?

চলো একদম সহজ একটা উদাহরণ দিয়ে শুরু করি।

উদাহরণ — রান্নার রেসিপি

তোমার মা যদি তোমাকে বলে — "ডিম ভাজো" — তুমি কী করবে?

প্রথমে তেল দাও। তারপর চুলায় দাও। তারপর ডিম ভাঙো। তারপর নাড়ো। তারপর নামাও।

এই ধাপগুলো মিলিয়েই হলো একটা রেসিপি — মানে, কী করতে হবে সেটার একটা নির্দেশনা।

প্রোগ্রামিং হলো ঠিক এটাই — কিন্তু রান্নার বদলে তুমি কম্পিউটারকে নির্দেশনা দিচ্ছো।

কম্পিউটার নিজে থেকে কিছুই করে না। সে শুধু সেটাই করে যা তাকে বলা হয়েছে। তুমি যদি বলো — "এই দুটো সংখ্যা যোগ করো এবং ফলাফল দেখাও" — সে ঠিক তাই করবে। এই নির্দেশনাগুলো লেখার কাজটাই হলো প্রোগ্রামিং।

Program কম্পিউটারকে দেওয়া ধাপে ধাপে নির্দেশনার সমষ্টি। ঠিক যেমন রেসিপি।
Programming সেই নির্দেশনা লেখার কাজ। যে লেখে, সে হলো Programmer।
Code কম্পিউটারের ভাষায় লেখা সেই নির্দেশনা। আমরা বাংলায় কথা বলি — কম্পিউটার "কোডে" কথা বোঝে।

এখন প্রশ্ন হলো — কম্পিউটার কোন ভাষায় কথা বোঝে? উত্তর হলো অনেক ভাষা আছে — Python, Java, C, C++, JavaScript... প্রতিটা ভাষার নিজস্ব নিয়ম আছে। আমরা শিখবো JavaScript।

পাঠ ০২

JavaScript কী এবং এটা কোথায় জন্ম নিয়েছে?

JavaScript-এর নাম শুনে অনেকে ভাবে এটা হয়তো Java নামের আরেকটা ভাষার মতো। কিন্তু না — এ দুটো সম্পূর্ণ আলাদা জিনিস। নামে মিল থাকলেও এরা আলাদা ভাষা। (ঠিক যেমন "বাংলাদেশ" আর "বাংলাদেশী মিষ্টি" দুটো আলাদা জিনিস হওয়া সত্ত্বেও নামে মিল আছে।)

🌐 JavaScript-এর জন্মের গল্প

সাল ছিলো ১৯৯৫। তখন internet সবে শুরু হয়েছে। Website ছিলো — কিন্তু সব website ছিলো মৃতের মতো স্থির। তুমি একটা পেজে গেলে, শুধু লেখা দেখতে পেতে। কোনো নড়াচড়া নেই, কোনো পরিবর্তন নেই।

Brendan Eich নামের এক ব্যক্তি মাত্র ১০ দিনে JavaScript ভাষাটা তৈরি করলেন যাতে website-কে জীবন্ত করা যায়। আজ সে ভাষাটাই দুনিয়ার সবচেয়ে বেশি ব্যবহৃত programming ভাষা।

সহজ কথায় বলতে গেলে:

ভাষা / জিনিস কী করে
HTML Website-এর কাঠামো বানায়। যেমন — একটা বাড়ির দেওয়াল, ছাদ, জানালা।
CSS Website-এর রং-সাজসজ্জা করে। যেমন — দেওয়ালে রং, পর্দা, আলো।
JavaScript Website-কে জীবন্ত ও স্মার্ট করে। যেমন — দরজা নিজে খোলা, বাতি জ্বলা, কেউ ডাকলে সাড়া দেওয়া।
সহজ উদাহরণ — মানুষের শরীর

একটা মানুষের শরীরের কথা ভাবো।

HTML হলো হাড় — শরীরের কাঠামো।
CSS হলো চামড়া, পোশাক — বাইরে কেমন দেখাবে।
JavaScript হলো মস্তিষ্ক ও পেশি — নড়াচড়া করা, চিন্তা করা, কাজ করা।

পাঠ ০৩

"Execute" মানে কী? Code কীভাবে কাজ করে?

তুমি যখন কোড লেখো, সেটা শুধু একটা Text File — মানে লেখা ভরা একটা ফাইল। কিন্তু সেই লেখা কীভাবে কাজ করে? কে সেটা বোঝে?

Execute বাংলায় মানে — "চালানো" বা "কার্যকর করা"।
যখন কম্পিউটার তোমার লেখা কোড পড়ে এবং সেই অনুযায়ী কাজ করে — সেটাই হলো code execute হওয়া।
Execution Execute করার পুরো প্রক্রিয়াটা। যেমন — রেসিপি পড়া থেকে শুরু করে রান্না শেষ করা পর্যন্ত পুরো কাজটাই হলো Execution।
Runtime কোড যখন চলছে (execute হচ্ছে), সেই সময়টাকেই বলে Runtime। মানে "চলার সময়"।
উদাহরণ — চিঠি পাঠানো

ধরো তুমি একটা চিঠি লিখলে। চিঠিটা লেখা হয়ে গেলো, কিন্তু এখনো কোনো কাজ হয়নি। তুমি যখন সেটা পোস্ট অফিসে দিলে এবং পোস্টম্যান সেটা পৌঁছে দিলো — তখনই কাজটা হলো।

কোডের ক্ষেত্রেও তাই:
— তুমি কোড লিখলে (চিঠি লেখা)
— কম্পিউটার সেটা পড়লো ও কাজ করলো (পোস্টম্যান পৌঁছে দিলো) = Execute হলো

এখন প্রশ্ন হলো — JavaScript কোড কে পড়ে এবং execute করে? এই কাজটা করে JavaScript Engine।

⚙️ JavaScript Engine কী?

JavaScript Engine হলো একটা বিশেষ প্রোগ্রাম যে তোমার JavaScript কোড পড়ে এবং সেটা কম্পিউটারকে বুঝিয়ে দেয়। কম্পিউটার সরাসরি JavaScript বোঝে না — Engine হলো মাঝখানের দোভাষী (Translator)।

সবচেয়ে বিখ্যাত JavaScript Engine হলো Google Chrome ব্রাউজারে থাকা V8 Engine।

পাঠ ০৪

JavaScript কোথায় চলে?

এই প্রশ্নটা অনেক গুরুত্বপূর্ণ। JavaScript দুটো জায়গায় চলতে পারে:

১

🌐 Browser-এ (ব্রাউজার)

Chrome, Firefox, Edge, Safari — এগুলো হলো Browser। এই Browser-গুলোর ভেতরে Built-in JavaScript Engine আছে। তুমি যখন Facebook, YouTube বা যেকোনো website খুলো — সেখানে JavaScript ব্রাউজারে execute হয়। এটাকে বলে Client-side (মানে তোমার কম্পিউটারের দিকে)।

২

🖥️ Server-এ (Node.js দিয়ে)

Node.js হলো একটা Tool যেটা দিয়ে JavaScript কে Browser-এর বাইরে, সরাসরি কম্পিউটারে চালানো যায়। এটা এখন অনেক বড় Topic। পরে একদিন শিখবে। এটাকে বলে Server-side।

আমরা এখন শুধু Browser-এ JavaScript চালাবো। কারণ এটাই সবচেয়ে সহজ এবং সবচেয়ে মজার।

Browser Chrome, Firefox, Edge ইত্যাদি। এগুলো দিয়ে আমরা Internet browse করি। এগুলোর ভেতরে JavaScript Engine আছে।
Client তোমার কম্পিউটার বা ফোন — যেটা website দেখছে সেটা। Client মানে "ব্যবহারকারীর দিক।"
Server দুনিয়ার কোথাও থাকা একটা শক্তিশালী কম্পিউটার যে website-এর তথ্য সংরক্ষণ করে ও পাঠায়।
উদাহরণ — রেস্টুরেন্ট

ধরো তুমি একটা রেস্টুরেন্টে গেলে।

তুমি (Client) — খাবার অর্ডার করো।
রান্নাঘর (Server) — খাবার তৈরি করে পাঠায়।
Browser — তোমার টেবিল, যেখানে খাবার এসে পৌঁছায়।
JavaScript — রেস্টুরেন্টের সব কিছু জীবন্ত রাখে: মেনু বোর্ড জ্বলে, অর্ডার নেওয়া হয়, বিল হিসাব হয়।

পাঠ ০৫

Console — Developer-এর গোপন জানালা

Browser-এ একটা লুকানো জায়গা আছে — শুধু Developer (মানে আমাদের মতো মানুষ) ব্যবহার করে। এটার নাম হলো Console।

Console Browser-এর ভেতরে একটা Text-based এলাকা যেখানে JavaScript আউটপুট দেখায়, error দেখায় এবং তুমি সরাসরি কোড চালাতে পারো। সাধারণ user এটা কখনো দেখে না। এটা শুধু Developer-দের জন্য।

Console খুলতে হলে Chrome-এ F12 চাপো, তারপর "Console" ট্যাবে ক্লিক করো। অথবা যেকোনো webpage-এ Right Click → Inspect → Console।

🎯

এখনই চেষ্টা করো! Chrome খুলে F12 চাপো। একটা কালো বা সাদা বক্স দেখবে — এটাই Console। এখানেই আমাদের প্রথম কোড চলবে।

পাঠ ০৬ — সবচেয়ে গুরুত্বপূর্ণ

তোমার প্রথম কোড — একটা একটা করে ভেঙে বুঝি

নিচে একটা JavaScript কোড দেখো। এখনো বুঝতে না পারলেও সমস্যা নেই — এরপর প্রতিটা অংশ একদম ভেঙে ভেঙে বুঝাবো।

প্রথম কোড — Browser Console-এ লেখো 
// আমার প্রথম JavaScript কোড

console.log("আমি JavaScript শিখছি!");

console.log("আমার নাম শিক্ষার্থী");

console.log(100 + 200);
Output (Console-এ যা দেখবে)
আমি JavaScript শিখছি!
আমার নাম শিক্ষার্থী
300

এখন চলো এই তিনটা লাইন একদম শুরু থেকে ভাঙি।

🔍 console মানে কী?

console হলো Browser-এর একটা Built-in Object। এখন তুমি "Object" কী সেটা না জানলেও সমস্যা নেই — পরে সেটা শিখবে। এখনকার জন্য মনে রাখো: console = Browser-এর সেই গোপন কালো বাক্স, যেটায় আমরা কিছু দেখাতে পারি।

🔍 . (dot/বিন্দু) মানে কী?

console.log — এখানে মাঝখানে একটা বিন্দু (.) আছে। এই বিন্দুর মানে হলো: "এর মধ্যে যা আছে সেটা ব্যবহার করো।"

যেমন: তোমার ব্যাগের ভেতরে যদি একটা কলম থাকে, তাহলে বলবে "ব্যাগ.কলম"। এখানে "ব্যাগ" হলো console আর "কলম" হলো log।

মানে: console-এর ভেতরে থাকা log নামের একটা জিনিস ব্যবহার করো।

🔍 log মানে কী?

log হলো একটা Function (কাজ করার যন্ত্র)। এখন "Function" কী সেটা পুরোপুরি না বুঝলেও চলবে — পরের ক্লাসে গভীরে যাবে। এখনকার জন্য মনে রাখো:

log মানে "দেখাও"। Console-কে বলছো: "এই জিনিসটা দেখাও।"

log শব্দের মানে হলো "রেকর্ড করো" বা "লিখে রাখো"। Console আমাদের দেখানোর জন্য জিনিসটা "লিখে রাখে" — তাই এর নাম log।

🔍 () এই দুটো বন্ধনী মানে কী?

console.log() — এই গোলাকার বন্ধনী দুটো হলো Function call করার চিহ্ন। মানে: "এই কাজটা এখন করো।"

যদি বলো console.log — এটা শুধু বলছে "আমি জানি log আছে।"
কিন্তু বলো console.log() — এটা বলছে "এখনই log-টা চালাও!"

আর বন্ধনীর ভেতরে যা দেবে — সেটাই সে দেখাবে। এটাকে বলে Argument বা Input। পরের ক্লাসে এটা ভাঙবো।

🔍 "আমি JavaScript শিখছি!" — এই উদ্ধৃতিচিহ্ন কেন?

JavaScript-এ যখন তুমি কোনো লেখা বা কথা লিখতে চাও, তখন সেটাকে দুটো Quotation mark (" ") বা (' ') দিয়ে ঘিরে দিতে হয়।

এই লেখাকে বলে String — মানে অক্ষর বা বর্ণের সারি। "String" শব্দের মানে সুতা — ঠিক যেমন সুতার মধ্যে অনেক তন্তু গাঁথা থাকে, String-এ অনেক অক্ষর একের পর এক গাঁথা থাকে।

উদ্ধৃতিচিহ্ন ছাড়া লিখলে JavaScript ভাববে এটা কোনো Special শব্দ — লেখা নয়। তখন Error আসবে।

তাই মনে রাখো: যেকোনো লেখা/Text লিখতে হলে — " " বা ' ' দিয়ে ঘিরে দাও।

🔍 শেষে ; (Semicolon) কেন?

বাংলায় একটা বাক্য শেষ হলে আমরা "।" দিই। ইংরেজিতে "." দিই।

JavaScript-এ একটা নির্দেশনা শেষ হলে ; দেওয়া হয়। এটা JavaScript-কে বলে: "এই নির্দেশনাটা এখানেই শেষ।"

আধুনিক JavaScript-এ Semicolon না দিলেও অনেক ক্ষেত্রে কাজ করে। কিন্তু ভালো অভ্যাস হলো প্রতিটা নির্দেশনার শেষে ; দেওয়া। এতে কোনো Confusion থাকে না।

🔍 // আমার প্রথম JavaScript কোড — এটা কী?

// দিয়ে শুরু হওয়া যেকোনো লেখাকে বলে Comment।

Comment হলো তোমার নিজের জন্য বা অন্য Developer-দের জন্য টীকা বা নোট। JavaScript এই লাইনটা পুরোপুরি উপেক্ষা করে — মানে Execute করে না।

এটা শুধু মানুষের পড়ার জন্য, কম্পিউটারের জন্য না। এটাকে বলে Single-line Comment।

🔍 100 + 200 — সংখ্যায় কেন " " নেই?

লক্ষ্য করো: console.log(100 + 200); এখানে সংখ্যার চারপাশে কোনো উদ্ধৃতিচিহ্ন নেই।

কারণ, লেখা (String) আর সংখ্যা (Number) দুটো আলাদা জিনিস। JavaScript-এ সংখ্যা সরাসরি লেখা যায়, কোনো " " লাগে না।

তুমি যদি লিখতে "100" + "200" — তাহলে Output হতো 100200 (দুটো লেখা জোড়া লাগানো)!
কিন্তু 100 + 200 লিখলে Output হয় 300 (আসল যোগফল)।

এটা Data Type-এর ব্যাপার — পরের ক্লাসে এটা নিয়ে গভীরে যাবো।

পাঠ ০৭

এই পুরো ব্যাপারটা একসাথে দেখি

এখন চলো দেখি — তুমি কোড লেখা থেকে শুরু করে Console-এ Output দেখা পর্যন্ত ঠিক কী কী হয়:

১

তুমি কোড লেখো

console.log("Hello"); — এটা এখন শুধু একটা লেখা। কোনো কাজ হয়নি। ঠিক যেমন রেসিপি বইয়ে লেখা আছে — কিন্তু রান্না হয়নি।

২

Browser-এর JavaScript Engine কোডটা পড়ে

Browser বলে: "আচ্ছা, এখানে console.log আছে। আমি জানি এটার মানে কী।"

৩

Engine কোড Execute করে

Engine কাজটা করে — Console-এ "Hello" দেখায়। এই মুহূর্তটাই হলো Execution।

৪

তুমি Console-এ Output দেখো

Console-এ লেখা আসে: Hello — কাজ শেষ!

আরেকটু বেশি কিছু — দেখো JS কতটা স্মার্ট 
// সংখ্যা নিয়ে খেলা
console.log(50 * 3);      // গুণ
console.log(100 - 40);    // বিয়োগ
console.log(20 / 4);      // ভাগ

// লেখা নিয়ে খেলা
console.log("আমার বয়স: " + 15);

// বিভিন্ন লেখা জোড়া লাগানো
console.log("JavaScript " + "অনেক " + "মজার!");
Output
150
60
5
আমার বয়স: 15
JavaScript অনেক মজার!

📌 লক্ষ্য করো: String + Number লিখলে JavaScript সংখ্যাটাকে লেখায় রূপান্তরিত করে জোড়া লাগায়। এটা Data Type Coercion — পরে এটাও শিখবে। এখন শুধু দেখো এটা কাজ করছে।

পাঠ ০৮

আজকে কী কী শিখলে — এক নজরে

Programming কম্পিউটারকে ধাপে ধাপে নির্দেশনা দেওয়া।
JavaScript দুনিয়ার সবচেয়ে জনপ্রিয় programming ভাষা যা website-কে জীবন্ত করে।
Execute/Execution কোড পড়ে কাজ করার প্রক্রিয়া। লেখা → কাজ = Execution।
JavaScript Engine Browser-এর ভেতরে থাকা দোভাষী যে JS কোড বোঝে। Chrome-এ এটা V8।
Console Browser-এর লুকানো Developer এলাকা। F12 দিয়ে খোলা যায়।
console.log() Console-এ কিছু দেখানোর নির্দেশনা। আমাদের প্রথম হাতিয়ার।
String লেখা বা Text ডেটা। " " বা ' ' দিয়ে ঘিরে লিখতে হয়।
Number সংখ্যা ডেটা। সরাসরি লেখা যায়। যোগ, বিয়োগ, গুণ, ভাগ করা যায়।
Comment (//) Developer-এর নোট। JavaScript এটা উপেক্ষা করে।
Semicolon (;) একটা নির্দেশনার শেষ চিহ্ন। বাংলার "।" এর মতো।

🔮 পরের ক্লাসে কী আসছে?

আজকে তুমি দেখলে JavaScript কীভাবে শুরু হয়। কিন্তু এটা তো সবে শুরু! পরের ক্লাসে শিখবে যেটা ছাড়া কোনো প্রোগ্রামই চলে না —

  • Variable কী? — কম্পিউটারের স্মৃতিতে জিনিস রাখার বাক্স। তুমি কোডে নাম দিয়ে যেকোনো তথ্য সংরক্ষণ করতে পারবে।
  • Data Types গভীরে — String, Number, Boolean (হ্যাঁ/না), আর আরও কিছু। প্রতিটা কেন আলাদা সেটা বুঝবে।
  • নামকরণের নিয়ম — Variable-এর নাম রাখতে কী কী নিয়ম মানতে হয়।
  • let, const, var — তিনটা শব্দ, তিনটা আলাদা মানে। কোনটা কখন ব্যবহার করবে।
